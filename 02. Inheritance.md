## Day 2: Inheritance in Java – Detailed Notes

Inheritance is a fundamental concept in Object-Oriented Programming that allows a class to inherit properties and methods from another class. It establishes an **IS-A** relationship between classes. Today, we'll explore inheritance using a `Vehicle` hierarchy with `Car` and `Bike` as subclasses.

---

### 1. What is Inheritance?

- **Inheritance** enables a new class (called **subclass**, **child class**, or **derived class**) to acquire the fields and methods of an existing class (called **superclass**, **parent class**, or **base class**).
- The subclass can also add its own fields and methods or override existing ones.
- In Java, inheritance is implemented using the `extends` keyword.
- Java supports **single inheritance** (a class can extend only one superclass) but allows multiple levels of inheritance (a class can extend another class that extends another, etc.).

**Key Benefits:**
- Code reuse – common logic is written once in the superclass.
- Establishes a natural hierarchy.
- Polymorphism – a subclass object can be treated as an object of its superclass.

---

### 2. IS-A Relationship

- The IS-A relationship means that a subclass is a specialized version of the superclass.
- For example: **Car IS-A Vehicle**, **Bike IS-A Vehicle**.
- This relationship is transitive: if `Bike` extends `Vehicle`, and `Vehicle` extends `Object`, then `Bike` IS-A `Object` as well.

**Syntax:**
```java
class Vehicle { ... }

class Car extends Vehicle { ... }
class Bike extends Vehicle { ... }
```

---

### 3. Basic Inheritance Example

Let's start with a simple `Vehicle` class that has common attributes and behaviors.

#### Vehicle.java
```java
public class Vehicle {
    private String brand;
    private String model;
    private int year;

    // Constructor
    public Vehicle(String brand, String model, int year) {
        this.brand = brand;
        this.model = model;
        this.year = year;
    }

    // Getters and Setters
    public String getBrand() { return brand; }
    public void setBrand(String brand) { this.brand = brand; }

    public String getModel() { return model; }
    public void setModel(String model) { this.model = model; }

    public int getYear() { return year; }
    public void setYear(int year) { this.year = year; }

    // Common behavior
    public void start() {
        System.out.println("Vehicle is starting...");
    }

    public void stop() {
        System.out.println("Vehicle is stopping...");
    }

    @Override
    public String toString() {
        return brand + " " + model + " (" + year + ")";
    }
}
```

#### Car.java (subclass)
```java
public class Car extends Vehicle {
    private int numberOfDoors;

    public Car(String brand, String model, int year, int numberOfDoors) {
        super(brand, model, year);  // call superclass constructor
        this.numberOfDoors = numberOfDoors;
    }

    public int getNumberOfDoors() { return numberOfDoors; }
    public void setNumberOfDoors(int numberOfDoors) { this.numberOfDoors = numberOfDoors; }

    // Car-specific method
    public void honk() {
        System.out.println("Car honks: Beep beep!");
    }

    @Override
    public void start() {
        System.out.println("Car engine roars to life...");
    }
}
```

#### Bike.java (subclass)
```java
public class Bike extends Vehicle {
    private boolean hasCarrier;

    public Bike(String brand, String model, int year, boolean hasCarrier) {
        super(brand, model, year);
        this.hasCarrier = hasCarrier;
    }

    public boolean hasCarrier() { return hasCarrier; }
    public void setHasCarrier(boolean hasCarrier) { this.hasCarrier = hasCarrier; }

    // Bike-specific method
    public void ringBell() {
        System.out.println("Bike bell rings: Ring ring!");
    }

    @Override
    public void start() {
        System.out.println("Bike is ready to pedal...");
    }
}
```

---

### 4. The `super` Keyword

- `super` is used to refer to the immediate parent class object.
- It can be used to:
  - Call the parent class constructor (`super(...)` must be the first statement in the subclass constructor).
  - Access parent class methods or fields that are hidden by the subclass.

**Example:** In `Car` constructor, `super(brand, model, year)` calls the `Vehicle` constructor to initialize inherited fields.

---

### 5. Method Overriding

- A subclass can provide a specific implementation of a method that is already defined in its superclass. This is called **method overriding**.
- The method in the subclass must have the same signature (name, parameter list) and return type (or a subtype – covariant return).
- Use `@Override` annotation to indicate overriding (optional but recommended for clarity and to catch errors).

In the example above, both `Car` and `Bike` override the `start()` method.

---

### 6. Access Modifiers in Inheritance

- **`private`** members are not inherited by subclasses.
- **`public`** and **`protected`** members are inherited.
- **`protected`** gives access within the same package and to subclasses, even if they are in different packages.
- **Package-private** (default) is inherited only if the subclass is in the same package.

**Tip:** To allow subclasses to access certain fields directly (without getters/setters), you can declare them as `protected`. However, it's often better to keep fields private and provide protected getters if needed.

---

### 7. Abstract Classes (Optional but Common in Hierarchies)

Sometimes a superclass is too generic to be instantiated. In such cases, we can declare it `abstract`. Abstract classes can have abstract methods (without a body) that must be implemented by subclasses.

**Example (using abstract Vehicle):**

```java
public abstract class Vehicle {
    // fields and concrete methods as before

    public abstract void move();  // abstract method – no body
}

public class Car extends Vehicle {
    @Override
    public void move() {
        System.out.println("Car is driving on road.");
    }
}
```

We'll keep our `Vehicle` concrete for simplicity, but in real design, `Vehicle` might be abstract.

### 8. File and Folder Structure

Organize your code in a package structure to avoid naming conflicts and to follow Java conventions. We'll use a Maven/Gradle style directory layout, but for simple learning, a flat structure with packages is enough.

**Recommended folder structure:**

```
Day02-Inheritance/
├── src/
│   └── com/
│       └── example/
│           └── vehicle/
│               ├── Vehicle.java
│               ├── Car.java
│               ├── Bike.java
│               └── Main.java
└── README.md (optional)
```

**Explanation:**
- `src/` contains source code.
- `com.example.vehicle` is the package name (reverse domain convention).
- Each class file is placed in the corresponding directory.

**Package declaration at the top of each file:**
```java
package com.example.vehicle;
```

**Compiling and running from the command line (inside `Day02-Inheritance` directory):**
```bash
javac -d out src/com/example/vehicle/*.java
java -cp out com.example.vehicle.Main
```

Or use an IDE like IntelliJ or Eclipse, which handles this automatically.

### 9. Demonstration with Main Class

#### Main.java
```java
package com.example.vehicle;

public class Main {
    public static void main(String[] args) {
        // Create a Car object
        Car myCar = new Car("Toyota", "Camry", 2022, 4);
        System.out.println("Car: " + myCar);  // toString() from Vehicle
        myCar.start();   // overridden in Car
        myCar.honk();    // Car-specific method

        System.out.println();

        // Create a Bike object
        Bike myBike = new Bike("Giant", "Escape 3", 2021, true);
        System.out.println("Bike: " + myBike);
        myBike.start();  // overridden in Bike
        myBike.ringBell();

        System.out.println();

        // Polymorphism: Vehicle reference pointing to Car object
        Vehicle vehicleRef = new Car("Honda", "Civic", 2023, 4);
        vehicleRef.start();  // Calls Car's start() – dynamic method dispatch
        // vehicleRef.honk(); // Compilation error – Vehicle doesn't know honk()
    }
}
```

**Output:**
```
Car: Toyota Camry (2022)
Car engine roars to life...
Car honks: Beep beep!

Bike: Giant Escape 3 (2021)
Bike is ready to pedal...
Bike bell rings: Ring ring!

Car engine roars to life...
```

### 10. Key Points to Remember

- **Single Inheritance:** Java class can extend only one class.
- **Constructors are not inherited,** but subclass constructor must call a superclass constructor (implicitly or explicitly). If not written, Java inserts a call to the default (no-arg) constructor of the superclass. If the superclass lacks a no-arg constructor, you must explicitly call `super(...)`.
- **`final` classes cannot be extended.**
- **`final` methods cannot be overridden.**
- **`static` methods are inherited but not overridden** (they can be redefined, but that's hiding, not overriding).
- Use `instanceof` to check the actual type of an object, but prefer polymorphism when possible.

### Summary

- Inheritance models IS-A relationships.
- Use `extends` to create a subclass.
- The `super` keyword accesses parent class members.
- Override methods to provide specialized behavior.
- Encapsulation still applies; keep fields private, provide protected/public accessors as needed.
- Proper package organization helps manage large projects.

Inheritance is a powerful tool for code reuse and establishing hierarchies. Next, we'll explore polymorphism in more depth and then move to design patterns.
