## Day 2 — Inheritance

Inheritance is one of the fundamental concepts of Object-Oriented Programming (OOP). It allows a class to inherit properties and methods from another class. The class that is inherited from is called the **parent class** (or superclass), and the class that inherits is called the **child class** (or subclass).

### IS-A Relationship

When a class inherits from another, it establishes an **IS-A** relationship. For example:

- A **Car** **IS-A** **Vehicle**.
- A **Bike** **IS-A** **Vehicle**.

This means that any object of the child class can be treated as an object of the parent class. The child class automatically gets all non‑private members of the parent, and it can also add its own specific members or override existing ones.

---

### Problem Statement

Design a simple inheritance hierarchy to represent different types of vehicles.

1. Create a base class called **Vehicle** with:
   - A `brand` attribute (String).
   - A constructor to initialize the brand.
   - A method `move()` that prints a generic message like `"The vehicle is moving."`.
   - A method `displayInfo()` that prints the brand.

2. Create a derived class **Car** that inherits from Vehicle. It should:
   - Have an additional attribute `doors` (int) representing the number of doors.
   - Provide a constructor to initialize both brand and doors.
   - Override the `move()` method to print `"The car is driving on the road."`.
   - Override `displayInfo()` to also show the number of doors.

3. Create another derived class **Bike** that inherits from Vehicle. It should:
   - Have an additional attribute `gears` (int) representing the number of gears.
   - Provide a constructor to initialize both brand and gears.
   - Override the `move()` method to print `"The bike is pedaling along the path."`.
   - Override `displayInfo()` to also show the number of gears.

4. In a main class, create objects of Car and Bike, and call their methods to demonstrate inheritance and polymorphism.

---

### Explanation of the Solution

- **Inheritance** is achieved using the `extends` keyword in Java. Both `Car` and `Bike` extend `Vehicle`, so they inherit the `brand` field and the methods `move()` and `displayInfo()`.
- **Constructor Chaining**: In each child class constructor, we call the parent constructor using `super(brand)` to initialize the inherited `brand` field.
- **Method Overriding**: The child classes override `move()` and `displayInfo()` to provide specific behavior. The `@Override` annotation is used (optional but recommended) to indicate overriding.
- **Polymorphism**: Because of the IS-A relationship, we could also store a `Car` or `Bike` object in a `Vehicle` reference, and the overridden methods would still be called (dynamic method dispatch).

---

### Java Solution

```java
// Base class Vehicle
class Vehicle {
    protected String brand;

    // Constructor
    public Vehicle(String brand) {
        this.brand = brand;
    }

    // Generic move method
    public void move() {
        System.out.println("The vehicle is moving.");
    }

    // Display brand
    public void displayInfo() {
        System.out.println("Brand: " + brand);
    }
}

// Derived class Car
class Car extends Vehicle {
    private int doors;

    // Constructor
    public Car(String brand, int doors) {
        super(brand);   // call parent constructor
        this.doors = doors;
    }

    // Override move()
    @Override
    public void move() {
        System.out.println("The car is driving on the road.");
    }

    // Override displayInfo() to also show doors
    @Override
    public void displayInfo() {
        super.displayInfo();   // call parent version to show brand
        System.out.println("Doors: " + doors);
    }
}

// Derived class Bike
class Bike extends Vehicle {
    private int gears;

    // Constructor
    public Bike(String brand, int gears) {
        super(brand);
        this.gears = gears;
    }

    // Override move()
    @Override
    public void move() {
        System.out.println("The bike is pedaling along the path.");
    }

    // Override displayInfo() to also show gears
    @Override
    public void displayInfo() {
        super.displayInfo();
        System.out.println("Gears: " + gears);
    }
}

// Main class to test the inheritance
public class InheritanceDemo {
    public static void main(String[] args) {
        // Create a Car object
        Car myCar = new Car("Toyota", 4);
        System.out.println("Car Info:");
        myCar.displayInfo();
        myCar.move();

        System.out.println();   // blank line

        // Create a Bike object
        Bike myBike = new Bike("Giant", 21);
        System.out.println("Bike Info:");
        myBike.displayInfo();
        myBike.move();

        // Demonstrating polymorphism
        System.out.println("\n--- Polymorphism demo ---");
        Vehicle vehicleRef;   // reference of type Vehicle

        vehicleRef = myCar;    // Car IS-A Vehicle
        vehicleRef.move();     // calls Car's move()

        vehicleRef = myBike;   // Bike IS-A Vehicle
        vehicleRef.move();     // calls Bike's move()
    }
}
```

**Output:**

```
Car Info:
Brand: Toyota
Doors: 4
The car is driving on the road.

Bike Info:
Brand: Giant
Gears: 21
The bike is pedaling along the path.

--- Polymorphism demo ---
The car is driving on the road.
The bike is pedaling along the path.
```

---

### Key Takeaways

- Inheritance creates an **IS-A** relationship.
- The `super` keyword is used to access parent class members.
- Overridden methods allow child classes to provide specialized behavior.
- Polymorphism lets us use a parent type reference to call the correct overridden method at runtime.

This is the essence of inheritance in Java. Practice by extending the hierarchy with more vehicle types (e.g., Truck, Bus) and adding more specific attributes and methods.
