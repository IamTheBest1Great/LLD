# Problem Statement — Design a `User` Class

Design a `User` class for a backend system with the following requirements:

### Fields

* `id` → unique identifier (must be positive)
* `name` → cannot be null or empty
* `email` → must be valid format and cannot be empty

### Functional Requirements

1. Fields must be **private** (encapsulation).
2. Provide a **constructor** to initialize all fields.
3. Provide **getters** for all fields.
4. Allow updating **name and email only** using setters.
5. `id` must be **immutable** (cannot be changed after creation).
6. Add **input validation** in constructor and setters.
7. Override `toString()` for readable output.

### Constraints

* Prevent invalid object creation.
* Follow clean OOP principles.

# Solution — User.java

```java
package com.lld.day01;

public class User {

    // encapsulated fields
    private final int id;     // immutable
    private String name;
    private String email;

    // constructor
    public User(int id, String name, String email) {
        validateId(id);
        validateName(name);
        validateEmail(email);

        this.id = id;
        this.name = name;
        this.email = email;
    }

    // getters
    public int getId() {
        return id;
    }

    public String getName() {
        return name;
    }

    public String getEmail() {
        return email;
    }

    // setters (no setter for id → immutable)
    public void setName(String name) {
        validateName(name);
        this.name = name;
    }

    public void setEmail(String email) {
        validateEmail(email);
        this.email = email;
    }

    // validation methods
    private void validateId(int id) {
        if (id <= 0) {
            throw new IllegalArgumentException("Id must be positive");
        }
    }

    private void validateName(String name) {
        if (name == null || name.trim().isEmpty()) {
            throw new IllegalArgumentException("Name cannot be empty");
        }
    }

    private void validateEmail(String email) {
        if (email == null || email.trim().isEmpty() || !email.contains("@")) {
            throw new IllegalArgumentException("Invalid email");
        }
    }

    // readable output
    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", email='" + email + '\'' +
                '}';
    }
}
```

# Test Class — Main.java

```java
package com.lld.day01;

public class Main {

    public static void main(String[] args) {

        User user = new User(1, "Shubham", "shubham@email.com");

        System.out.println(user);

        user.setName("Shubham Kumar");
        user.setEmail("sk@email.com");

        System.out.println(user);

        // user.setId(5); ❌ not allowed (immutable)
    }
}
```

# Why this is the correct professional approach

Bad beginner version:

```
public int id;
public String name;
```

This is **trash design** because:

* No encapsulation
* No validation
* No immutability
* Any code can corrupt state

Good version (above):

* Controlled mutation
* Immutable identity
* Safe object creation
* Interview-ready
